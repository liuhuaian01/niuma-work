# 记忆页面优化方案 + 太极引擎前端映射分析报告

## 一、当前记忆页面状态

### 已有功能（已实现）
- ✅ 三段式 Tab 导航：日记 / 做梦 / 长期记忆
- ✅ 日记：日历组件 + Agent/项目列表 + 自动化/当日任务展示
- ✅ 做梦：Agent 日记卡片网格 + 详情抽屉（点击卡片展开）
- ✅ 长期记忆：左侧 Agent/项目分类列表（已有 HTML 结构）
- ✅ 日历翻页、日期选择、Agent 筛选

### 待优化/缺失
- ❌ CSS 颜色变量未对齐设计系统 v4.0 Token（用了旧版绿色/紫色主题）
- ❌ 卡片样式未使用 `--bg-card`/`--border` 等新设计 Token
- ❌ 长期记忆 Tab 内容面板不完整（缺少记忆卡片展示）
- ❌ 缺少记忆搜索功能
- ❌ 缺少记忆统计（总条数、今日新增、最近审查次数）
- ❌ 缺少记忆编辑/删除交互（当前仅静态展示）
- ❌ 日记页面缺少"写日记"入口
- ❌ 无太极引擎状态可视化

## 二、优化方案（结合设计系统 + 太极引擎 + 审美提升）

### 2.1 布局调整

```
┌──────────────────────────────────────┐
│  Page Header: 记忆                    │
│  统计条：总条数 | 今日新增 | 最近审查  │
├──────────────────────────────────────┤
│  Tab导航: [日记] → [做梦] → [长期记忆] │
├──────────────────────────────────────┤
│                                      │
│  日记 Tab:                            │
│  ┌──────┬─────────────────────────┐  │
│  │ 日历  │  选中日期标题            │  │
│  │ 卡    │  ┌──────────────────┐  │  │
│  │ 片    │  │ 自动化任务列表     │  │  │
│  │      │  │ + 当日任务列表     │  │  │
│  │      │  │ + 写日记入口        │  │  │
│  │      │  └──────────────────┘  │  │
│  └──────┴─────────────────────────┘  │
│                                      │
│  做梦 Tab（当前结构可保留，优化样式）    │
│  卡片网格 → 统一使用设计系统 Token      │
│                                      │
│  长期记忆 Tab                          │
│  ┌──────┬─────────────────────────┐  │
│  │专家   │  记忆卡片时间线           │  │

│  │列表   │    内容摘要/操作按钮     │  │
│  │(同日记)│  - 支持搜索/过滤         │  │
```

### 2.2 CSS Token 对齐（需要修改的颜色对照表）

| 当前值 | 应改为 | 说明 |
|--------|--------|------|
| `rgba(77,168,240,0.12)` | `var(--brand-muted)` | 品牌色浅底 |
| `#4DA8F0` | `var(--brand)` | 品牌主色 |
| 硬编码绿色/紫色 | `var(--accent)` / `var(--coral)` | 语义色 |
| 硬编码阴影 | `var(--shadow-sm)` / `var(--shadow)` | 阴影系统 |
| 硬编码圆角 | `var(--r-md)` / `var(--r-lg)` | 圆角系统 |

### 2.3 需增加的功能

1. **顶部统计条**（日记 Tab 上方）
   - 记忆总条数 | 今日新增 | 最近审查次数 | 最后一次做梦时间
   - 使用设计系统 `--fs-xl` 显示数字

2. **记忆搜索框**（全局搜索所有记忆）
   - 统一使用 `.page-search` 样式（已有定义）
   - 搜索范围：日记内容、记忆卡片、任务描述

3. **"写日记"入口**
   - 在日记页面右侧底部或顶部，添加一个"写日记"按钮
   - 点击弹出模态框，输入内容后保存到时间线

4. **长期记忆 Tab 完整实现**
   - 当前左侧列表已有，但右侧内容区缺少记忆卡片
   - 需要渲染三分类卡片：用户画像 / 项目约定 / 安全策略
   - 每张卡片：时间戳 + 内容摘要 + 编辑/删除按钮
   - 支持按类型筛选

5. **记忆编辑/删除交互**
   - 日记条目 hover 显示编辑/删除按钮
   - 长期记忆卡片直接显示操作按钮
   - 删除前确认弹窗

### 2.4 审美提升（Taste Skill 应用）

1. **卡片设计升级**
   - 减少边框厚重感：仅用 1px `var(--border)` 细线
   - 悬停效果：`translateY(-2px)` + 边框变品牌色
   - 内边距统一 16px（`p-4`）

2. **日历组件美化**
   - 当前日历 6x7 网格，每个日期格 32×32px，偏小
   - 改为 `gap:2px`，日期格 36×36px，字体 13px
   - 选中日期的圆点从纯色改为渐变品牌色

3. **任务列表精简**
   - 每项高度统一 40px
   - 状态圆点 8×8px（正常 6px）
   - 标签（自动化/手动）使用 `12px` 字号 + `6px` 圆角

4. **过渡动画**
   - Tab 切换：淡入淡出 150ms（非 300ms，保持轻快）
   - 卡片悬停：200ms ease
   - 日历日期悬停：底色变化

## 三、太极引擎在前端的体现

阅读了 `taiji-engine-architecture.md`（七律引擎架构文档），以下概念**应该在前端有视觉对应**：

### 3.1 必须体现的（P0）

**1. 进化循环可视化（进化进度环/图）**
- 记忆页应体现"生生不息"——展示记忆系统的进化状态
- 建议：在记忆页顶部或长期记忆 Tab 中添加一个**进化状态环**（类似健身 App 的圆环）
  - 今日进化次数 / 本周进化趋势
  - 记忆审查次数 / 技能沉淀数
  - 数据来源：后端 `evolution/status` API

**2. 三级记忆架构可视化**
- 在记忆页任一 Tab 中，用视觉区分 L1/L2/L3 层级
- 日记 = L1（会话记忆）的基础
- 长期记忆 = L3（知识库）
- 中间应该有一条"记忆温度"指示线（从短期→长期的可视化过渡）

### 3.2 应该体现的（P1）

**3. Hermes 注意力状态指示器**
- 记忆页顶部或全局导航中，显示 Hermes 当前状态：
  - 🟢 观察中（默认）——"以静制动"
  - 🟡 正在思考
  - 🔴 主动介入
- 体现"不是话痨，是狙击手"的理念

**4. 动态均衡指示器（阴阳平衡）**
- 在记忆页或办公室页显示当前模型使用比例
- 滑块/进度条：左端"本地（阴）"右端"云端（阳）"
- 中间滑块指示当前平衡点

### 3.3 推荐的（P2）

**5. 自愈回路日志**
- 在长期记忆 Tab 中，增加"系统自愈日志"类别
- 展示最近的自愈事件（拦截→替代方案→规则沉淀）
- 体现"用户不知道 Hermes 做了什么，但系统就是不出问题"
