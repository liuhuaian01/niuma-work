# 超级牛马 — 原型 vs Vue开发 深度差距分析

> 基准：`frontend/niuma-neon-pulse-prototype.html` (19,727行) vs `frontend-vue/src/` (52源文件)  
> 日期：2026-06-30 | 评估人：画统筹 (Hua)

---

## 总体判断

| 维度 | 原型完成度 | Vue完成度 | 差距 |
|:---|:--:|:--:|:--:|
| 页面/路由 | 8页 | 7页+Login | 差 1 关键页(设置) |
| 布局框架 | 100% | 90% | 三栏布局框架基本对齐 |
| 设计令牌 | 100% | 85% | Day/Classic主题部分缺失CSS |
| 对话交互 | 100% | 55% | 缺流式动画/思考指示/搜索/右键 |
| 项目模块 | 100% | 35% | 缺项目详情4个子标签 |
| UI组件库 | ~25组件 | ~15组件 | Toast/全局搜索/复杂Modal等未实现 |
| 动画系统 | 丰富 | 基础 | 缺交错动画/骨架屏/页面过渡 |

**综合差距评估：原型 UX 覆盖 Vue 现阶段的约 55%。**

## 一、页面级差距

### 1.1 设置页（P0 — 完全缺失）
> 原型有 5 大区块的设置页，Vue **完全不存在**。

| 设置区块 | 原型 | Vue |
|:---|:--:|:--:|
| API Keys 管理 | 3 提供商(DeepSeek/Kimi/Hunyuan)+新增按钮+测试连接 | ❌ |
| 语音设置 | 语音模型选择+语速/音调调节 | ❌ |
| 快捷键一览 | 搜索+类别分组表格 | ❌ |
| 关于信息 | 版本号+引擎状态+版权 | ❌ |
| 系统偏好 | 主题切换+系统设置入口 | 部分（仅主题切换在NavRail） |

**建议**：新建 `SettingsView.vue`，分 4 个 tabs（API Keys / 偏好 / 快捷键 / 关于）。

### 1.2 项目详情页（P0 — 存在框架，缺核心交互）
> 原型有 `pageProjectDetail`，内含 4 个子 Tab（Activity/SOP/Tasks/Assets）及内嵌项目聊天，Vue 的 ProjectsView **只实现了工作间列表+Kanban**。

| 缺失子页面 | 原型功能 |
|:---|:---|
| 项目 Activity 流 | SSE 对话流式输出 + 暂停/发送 + 消息渲染 |
| 项目 SOP 页 | 标准流程视图（静态卡片） |
| 项目 Tasks 页 | 任务列表视图 |
| 项目 Assets 页 | 资源管理 |
| 项目配置面板 | Connector/Expert/Skill 选择器（3 个弹出菜单 + 列表追加） |
| 项目 Agent 树 | 分组 + Agent 层级选择 |

**建议**：新建子路由 `/projects/:id`，实现 `ProjectDetailView.vue`。优先级：Activity > 配置 > Tasks > SOP > Assets。

### 1.3 首页（P2 — 原型有，已被合并）
> 原型有 `pageHome`（与 Chat 并存），Vue 已合并到 `/chat`。本项目不需要单独首页。**无差距需处理。**

## 五、设计系统与视觉差距

### 5.1 Token 对齐度：85%
- Vue `design-tokens.css` 对 Night 模式基本对齐原型
- Day/Classic 主题的 CSS 变量在 Vue 中**仅定义了粗略框架**，缺少原型中大量主题特定覆盖
- 原型有约 1800 行主题覆盖 CSS，Vue 对应覆盖**不到 200 行**

### 5.2 动画系统差距
| 动画 | 原型 | Vue |
|:---|:--:|:--:|
| 页面过渡 (fade+slide) | ✅ | ✅ Transition |
| Stagger 卡片渐入 | ✅ 每个卡片独立延迟 | ❌（CSS 无 stagger） |
| Nav 按钮品牌呼吸动画 | ✅ brandPulse | ✅ |
| 消息淡入 | ✅ | ❌ |
| Modal 缩放淡入 | ✅ | ✅ |
| Toast 滑入 | ✅ | ❌ |
| 骨架屏脉冲 | ✅ | ❌ |
| 实验室引擎光效 | ✅ animated gradient | ✅ |

### 5.3 主题切换
- 原型：三主题循环 + localStorage 持久化 + 图标切换 + Toast 提示
- Vue：三主题 + localStorage，基本对齐，但缺少 Toast 提示

## 九、风险提示

1. **对话模块** 是差异化最大的区域，原型有大量细节（骨架屏、思考指示、队列区、语音动画等），一刀切全实现会拉长周期。建议先做骨架屏+消息动画+右键菜单这 3 个最高频的。
2. **项目详情页** 涉及子路由、组件层级重构，建议先确认是否真有用户需要——如果现阶段项目页的使用方式就是工作间切换+Kanban，可能不需要完整的项目详情。
3. **主题 Day/Classic** 在原型中是完整实现的，但 Vue 只做了框架，补齐需要大量 CSS 重写，建议只补 Day 主题，Classic 可延后。
4. **文件预览/编辑器** 原型有但功能定义模糊，建议等有明确需求再做。
