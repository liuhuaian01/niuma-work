## 二、关键交互规范

### 导航
- **左侧垂直 Tab**（非顶部水平）
- `Cmd/Ctrl + F` 搜索框自动聚焦
- 实时过滤，300ms 防抖，匹配高亮 + Tab 数字角标
- 键盘导航：↑↓ 选择，Enter 跳转

### 即时生效 vs 需重启
- 外观类：即时生效，无弹窗
- AI 行为类：即时生效，下一条消息起应用
- 连接类：验证后立即可用
- 账户类：即时生效，安全等级切换需二次确认
- **原则：不要求用户重启应用**

### 恢复默认
- 每项独立恢复默认（悬停显示图标）+ `Cmd+Z` 撤销（最近 5 次操作）
- **不提供"全部重置"按钮**——全局重置是流失诱因

### 修改标记
- 蓝色小圆点（6px，`#3B82F6`）出现在设置项标签左侧
- Tab 级别也显示角标
- 恢复默认后立即消失

### 渐进式披露（3 层）
| 层级 | 名称 | 入口 | 覆盖 |
|:----:|------|------|------|
| L1 | Onboarding 向导 | 首次启动自动触发 | 80% 设置需求 |
| L2 | Quick Settings 面板 | 主界面齿轮图标 / 系统托盘 | P0 四项 + 场景切换 |
| L3 | 完整设置页 | `Cmd+,` / 齿轮 → "更多设置" | < 20% 用户进入 |

### 禁区词汇（永远不出现）
Token / 降级链 / 能力开关 / 太极引擎 / Hermes / API endpoint / 上下文窗口 / 推理链 / RAG / 温度

## 三、Onboarding 流程

### 3 步向导

**第 1 步：外观设置**
- 主题选择（浅/深/跟随系统，3 张大卡片）
- 语言自动检测，支持手动更改
- 始终可"跳过"

**第 2 步：AI 默认模型**
- 推荐模型置顶（🌟标记 + 一句话说明）
- 可选模型 ≤ 5 个，每个附简短描述

**第 3 步：API Key（可选）**
- 明确标记"可选——内置模型已经够用"
- 提供输入框 + 验证按钮
- "不知道这是什么？跳过，以后随时可以配置"

**设计原则**：每步 ≤ 3 个决策点；始终可跳过，最后显示推荐默认确认页；完成后直接进入主界面

### 主界面"一键三连"

位于主界面右下角：外观（单击切换）/ 语言（单击展开）/ AI 方向（单击展开场景列表）

快捷键：`Cmd+Shift+T`(主题) / `Cmd+Shift+L`(语言) / `Cmd+Shift+M`(模式)

## 四、分阶段交付计划

### v1.7 — Quick Settings 面板（P0，10 周）

**定位**：最小化设置入口，消灭 80% 设置需求

**交付物**：
- Onboarding 3 步向导
- 主界面"一键三连"
- Quick Settings 面板（Popover 形式，P0 四项）
- 所有设置即时生效

**Sprint 拆解**（4 个 Sprint）：
- S1.7.1（2 周）：Onboarding 骨架 — 向导框架 + 第 1-2 步
- S1.7.2（2 周）：Quick Settings 面板骨架 — API Key 验证 + 配置存储层
- S1.7.3（2 周）：P0 四项 + 一键三连
- S1.7.4（2 周）：集成测试 + 埋点 + 上线

**完成标准**：Onboarding 完成率 > 90%，首次配置时长 < 60s，面板打开 < 200ms

### v1.8 — 独立设置页（P0+P1，18 周）

**前置条件（三元组合）**：累计 5,000 对话 + v1.7 上线 ≥ 4 周 + 7 项埋点均有 ≥ 30 数据点
**硬性时间上限**：v1.7 上线后 12 周内启动 v1.8 评审，无论条件是否完全满足

**交付物**：
- 完整 7 Tab 设置页（左侧垂直 Tab）
- 搜索（设置项 > 30 强制启用）
- 技能管理 GUI（卡片列表 + 开关 + 配置入口）
- Agent 团队可视化配置
- 场景模式（创建/切换/管理）
- 修改标记蓝点系统

**Sprint 拆解**（6 个 Sprint）：
- S1.8.1（2 周）：设置页架构 — Tab 导航 + 通用组件库 + 搜索基础设施
- S1.8.2（2 周）：AI 偏好 + 外观 + 编辑器 三个 Tab
- S1.8.3（2 周）：连接 + 账户 + 关于 三个 Tab
- S1.8.4（3 周，XXL）：**技能 Tab** — 技能列表 + Agent 团队配置（全阶段最复杂）
- S1.8.5（2 周）：场景模式 + 搜索完善 + 修改标记 + 恢复默认
- S1.8.6（2 周）：集成测试 + 数据迁移 + 上线

**完成标准**：设置放弃率 < 20%，搜索依赖率 < 50%，技能管理访问率 ≥ 30%

### v2.0 — 完整体验（P2，12 周）

**交付物**：跨设备同步 / 第三方集成 / 技能使用报告 / 数据导出 / 自定义 Quick Settings / 开发者模式

## 五、竞品对标结论

| 竞品 | TAB 数 | 对我们的启示 |
|------|:-----:|-------------|
| ChatGPT | 5 | AI 独立 Tab（Personalization），不埋通用中 |
| Cursor | 7 | 左侧 Tab + 搜索驱动，开发者最佳实践 |
| Linear | ~8 | 极致克制，每 Tab < 10 项，修改视觉标记 |
| VS Code | 9 大类 | 搜索驱动型标杆，`@modified` 过滤器 |
| Notion | ~20 | 反面教材：双层侧栏太深太散 |
| Windsurf | 6 | AI 独立 Tab + 用量独立展示 |

**关键洞察**：技能管理 GUI **无竞品有等价功能**，是超级牛马独有差异化。

## 六、用户研究关键洞察

| 用户画像 | 核心心态 | 对设置页的启示 |
