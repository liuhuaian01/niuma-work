## 1. 产品概述

### 1.1 一句话定义

**超级牛马工作台 = 太极引擎驱动的 AI Agent 协作平台。**

底层核心引擎：太极引擎（Taiji Engine）——52 个引擎模块，七元驱动的自动进化系统。
底层框架：Hermes Agent v0.16.0（MIT 开源，170+ 贡献者）→ 太极引擎正在容纳化。
桌面壳：WebView2 + FastAPI 后端。

用户通过自然语言与平台助手对话，太极引擎自动编排多个 Sub-Agent 完成复杂任务。本地优先，云端可选。独有 Token 管控体系——能力开关 + Agent 维度预算 + 降级链——竞品无一具备。

### 1.2 核心价值主张

| 维度 | 传统 AI 工具 | 超级牛马 |
|------|:----------:|:--------:|
| 部署 | 云端 SaaS，数据上传 | **本地优先，云端可选** |
| 交互 | 单轮对话，复制粘贴 | **多 Agent 协作，Swarm 编排** |
| 记忆 | 无状态，每次从零开始 | **三级记忆系统 + 用户画像，越用越懂你** |
| Token 管控 | 无控制，事后看账单 | **能力开关 + 预算 + 降级链 + CCR 压缩，全链路管控** |
| 扩展 | 插件市场，手动安装 | **MCP 工具接入 + Skill 自生成** |
| 算力 | 单机独享 | **太极网格 P2P 算力共享（v1.1）** |
| 成本 | 按 Token 计费，不可控 | **国产模型优先，Smart Allocator 自动推荐** |

### 1.3 产品边界（Pi 原则 —— 不该做的一律不做）

**前期不做**：
- 用户自定义 Agent 可视化编排
- 技能市场（社区分享）
- 云端同步/多设备协作
- 移动端 App
- FTP/云存储挂载

**前期聚焦**：
- RAG 知识库 → L3 向量检索 → 上下文压缩 → 太极网格算力共享

## 2. 核心设计哲学

### 2.0 太极产品哲学（根基）

超级牛马的产品 DNA 源于中国道家哲学——"道生一，一生二，二生三，三生万物"的生成论。

**道 = "更聪明地工作，而非更费力"** ——这是所有产品决策的本源判断标准。

| 原理 | 产品落地 |
|:--|:--|
| 阴阳平衡 | 本地+云端双模 · 平台全权限+被动行使 |
| 四两拨千斤 | Token 管控体系——不是加资源，是精准用力 |
| 以静制动 | 平台助手只在 5 种情况说话 · 静默编排 |
| 无为而治 | Pi 准则克制设计 · 前期不做列表 |
| 顺势而为 | 站在开源模型生态肩上 |
| 刚柔并济 | Capability Flags 硬开关 + 记忆闭环软积累 |
| 生生不息 | 递归自进化 + GoalLoop 闭环学习——越用越强 |

### 2.1 七元体系（v1.7 确立）

太极引擎从"七律→七引擎"升华为"七元体系"：

| 七元 | 核心组成 | 状态 |
|:--|:--|:--:|
| 平台三元 | Hermes Agent → Swarm 编排 → 工作间沙盒 | ✅ |
| 记忆三元 | L1 会话 → L2 短期档案 → L3 知识库 → Honcho 用户画像 | ✅ |
| 管控三元 | 能力开关 → Token 预算 → 降级链 | ✅ |

| 认知三元 | 个体学习 → 群体涌现 → 跨模块协同 | ✅ v1.0 |

### 2.2 11 条产品设计开发铁则

> 违反任意一条，视为设计 Bug。

| # | 原则 | 一句话 | 当前状态 |
|:--:|:--|:--|:--|
| 1 | **极致安全** | 数据不出本地 + 代码不可被破解 | 引擎身份绑定 + 回传匿名 SHA256 |
| 2 | **极致性能** | 快到你感觉不到 | Gemma-4 8GB 笔记本可运行 |
| 3 | **极致轻量** | 小、省、不打扰 | 52 引擎模块，零冗余 |
| 4 | **极致 Token 低消耗** | 每一分钱花刀刃上 | CCR 压缩层 + Smart Allocator + 降级链 |
| 5 | **极致高效** | 最少步骤完成任务 | pre_chat_check 并行检测 |
| 6 | **极致简单** | 奶奶都能用 | Auto 模式默认，零概念设计 |
| 7 | **极致质量** | 交付物可靠 | 质量门禁 + 递归自进化 |
| 8 | **极致审美** | 干净不脏 | Neon Pulse v10.0 · 三主题引擎 · 牛马蓝 |
| 9 | **极致体验** | 无打断感 | 流式 SSE，TTFB < 200ms |
| 10 | **极致交互** | 所见即所得 | 弹窗最小化，纯键盘可达 |
| 11 | **极致克制（Pi 原则）** | 不该做的坚决不做 | 该做的一件不少，不该做的一件不多 |

### 2.3 冲突仲裁规则

```
1. 安全 — 数据泄漏不可逆
2. 性能 — 慢 0.5 秒用户就走了
3. 轻量 — 每一行代码都有存在理由
4. 低 Token — 国产模型优先，CCR 自动压缩
5. 高效 — 先让用户上手，再让他们快
6. 简单 — Auto 模式零学习成本
7. 交互 = 质量 — 两者同等，一个差则全差
```

| 情况 | 示例 | 触发条件 |
|------|------|:--|

| 总结 | "已完成。要点：1... 2... 3..." | 任务完成后 |

## 3. 目标用户

### 3.1 核心用户画像

| 画像 | 描述 | 核心痛点 | 使用场景 |
|------|------|:--|:--|
| **独立创作者** | 网文写手、自媒体博主、设计师 | 写作卡壳、素材整理耗时、多平台分发繁琐 | 网文创作、文案生成、视频脚本 |

   └────────────────── 记忆沉淀 + 递归自进化，越用越强 ←──────────────────┘

## 4. 产品架构

### 4.1 整体架构图

**六层架构（v1.7 确立）**：

| 层级 | 核心组件 | 说明 |
|------|------|------|
| **基座层** | Hermes Agent（Root Node） → 太极引擎容纳化 | 预装运行，闭环学习，正在从寄生→共生→容纳 |
| **管控层** | Token 管控（超级牛马独有） | 能力开关、Agent 预算、降级链、外网访问控制 |
| **编排层** | Swarm 拓扑 | Root → Worker → Gate Validator → Gate Synthesizer |
| **平台层** | 前端 UI + 后端服务 + 安全中间件 | FastAPI + SQLite + LanceDB + 7 中间件 |
| **记忆层** | 三级记忆 + 用户画像 | L1 会话 + L2 短期档案(FTS5) + L3 知识库(LanceDB) + Honcho |
| **模型层** | 预装 + 广场 + 云端 API + 太极网格 | 本地部署(Ollama) + 云端可选 + 降级链 + P2P 算力 |

### 4.3 工作间隔离机制

| 资源 | 隔离方式 | 实现 |
|------|:--:|------|
| 任务 | 工作间内独立队列 | 后端 workspace_isolation 中间件 |
| 记忆 | 工作间作用域 | 分级存储，Sub-Agent 仅访问本工作间 |
| 文件 | 工作间目录隔离 | 每工作间独立文件空间 |
| Agent 通信 | 同工作间内直接通信 | 跨工作间必须经平台助手中转 |
| 上下文 | 独立对话上下文 | Sub-Agent 不可访问其他工作间的对话历史 |

## 9. 质量门禁与测试策略

### 9.1 质量门禁

```
