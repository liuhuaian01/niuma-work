# 超级牛马工作台 · 用户体验架构 v2.0

> 架构师：ArchitectUX | 日期：2026-06-29 | 基于 v1.7 审计结果重设计
> 本方案解决 6 项致命 UX 断点，定义统一的导航、流程、交互、状态管理规范

---

## 一、审计结论（6 项致命断点）

从深度审计中提取的 UX 层面核心问题：

| # | 断点 | 当前状态 | 影响面 |
|:--:|:--|:--|:--|
| 1 | **无端到端用户链路** | 用户无法完成"打开→说话→收到回复" | 产品不可用 |
| 2 | **独立页面无法互达** | Kanban/Token仪表盘是独立HTML，无导航连接 | 功能孤岛 |
| 3 | **引导流程缺失** | UX Spec定义了3步引导，原型中不存在 | 新用户迷失 |
| 4 | **品牌色6版本矛盾** | UX Spec薄荷绿 vs 原型牛马蓝 vs DS v8蓝紫 | 设计分裂 |
| 5 | **设计系统不可引用** | CSS全内联在HTML中，无独立link标签 | 无法复用 |
| 6 | **20+后端能力无前端** | 仅Chat/Workspace/Health/Models有前端集成 | 能力沉睡 |

## 二、架构决策（必须遵守的 8 条铁则）

### 决策 1：品牌色 → 牛马蓝 #4DA8F0（一锤定音）

**理由**：所有实际实现（原型、Kanban、Dashboard、DS v7-v8、API客户端）全部使用 `#4DA8F0`。UX Spec v1.0 的薄荷绿 `#10B981` 是孤例，从未落地。

**执行**：UX Spec v1.0 品牌色章节更新为牛马蓝，删去"禁紫色"条款（DS的紫色仅用于强调色，非品牌主色）。

### 决策 2：主题系统 → Day 为默认，Night 为暗色选项

**理由**：
- UX Spec 规定"主题：浅色（仅浅色）"——但用户可能需要暗色
- 采用三按钮切换：Light / Dark / System，默认跟随系统
- DS v16.0 已有完整的 Day/Night/Classic 三主题种子

### 决策 3：导航结构 → 沿用现有 2 栏设置布局，增加"工作台"分组

基于现有前端原型 `niuma-neon-pulse-prototype.html` 的设置页布局：

```
设置页（2 栏布局，复用现有 CSS）
├── 左侧：Settings Nav Card（260px）
│   ├── 通用
│   │   ├── 偏好设置
│   │   ├── 个人资料
│   │   ├── 系统设置
│   │   ├── 语音输入
│   │   └── 快捷键
│   ├── 工作台          ← 新增分组
│   │   ├── 任务看板    ← Kanban 移入
│   │   └── 用量看板    ← Token 仪表盘移入
│   ├── 高级设置
│   │   ├── 安全中心
│   │   ├── 意识
│   │   ├── 太极引擎
│   │   ├── 已归档
│   │   └── 关于
│
├── 右侧：Settings Detail Card（flex:1）
│   每个 nav item 对应一个 settings-panel
│   Kanban 和仪表盘各自封装为一个 settings-panel
│   通过 N.switchSetting() 切换，与现有交互一致
```

**关键变化**：
- Kanban 和 Token 仪表盘从独立 HTML 文件移入设置页的 detail card
- 不新增一级页面，不破坏现有布局，零布局重构
- 左侧 nav card 新增"工作台"分组，包含 2 项
- 右侧每个对应一个 `settings-panel`，与已有 panel 切换机制一致

### 决策 4：设计系统引用 → 必须通过 link 标签引入

所有 HTML 页面强制：
```html
<link rel="stylesheet" href="css/design-tokens.css">
<link rel="stylesheet" href="css/components.css">
```
禁止内联 `<style>` 中重复定义 Token。

### 决策 5：对话链路 → 必须 3 步内完成

```
打开 → 输入框自动聚焦 → 输入文字 → 按Enter → SSE 流式回复
```

零障碍：不弹引导（已有工作间则跳过），不弹模型选择器（默认Auto），不弹配置。

### 决策 6：前端入口 → 单页应用（SPA）路由

使用 Vue Router（hash 模式），所有页面在同一 HTML 内切换，无整页刷新。

### 决策 7：响应式 → 1280×720 为最低分辨率

桌面优先，不做移动端适配。1280px 以下隐藏次要面板。

### 决策 8：空状态 → 必须含引导操作

每个空状态必须包含：文案说明 + 引导按钮/快捷入口。不出现"暂无数据"后无下一步的空白页。

## 五、状态管理矩阵

每个页面必须覆盖以下状态：

| 状态 | 对话 | 工作室 | 广场 | 记忆 | 连接 | 设置 |
|:--|:--:|:--:|:--:|:--:|:--:|:--:|
| **加载中** | 骨架屏 | 骨架屏 | 骨架屏 | 骨架屏 | Spinner | 空 panel + 骨架 |
| **空状态** | 引导消息 | 创建按钮 | — | 提示文案 | 配置按钮 | 看板 panel 空状态 |
| **正常** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **错误** | Toast+重试 | Toast | Toast | Toast | Toast | 看板 panel Mock 回退 |
| **离线** | 本地模式 | 本地缓存 | 缓存数据 | 本地数据 | 断连图标 | 看板 panel 演示数据 |
| **首次使用** | 3步引导 | 创建引导 | — | — | — | 偏好 panel 默认 |

### 5.1 骨架屏规范

```css
.skeleton {
  background: linear-gradient(
    90deg,
    var(--color-bg-subtle) 25%,
    var(--color-bg-hover) 50%,
    var(--color-bg-subtle) 75%
  );
  background-size: 200% 100%;
  animation: skeleton-shimmer 1.5s infinite;
  border-radius: var(--radius-sm);
}

@keyframes skeleton-shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}
```

应用到：消息列表、卡片列表、表格、图表区域。

### 5.2 空状态组件规范

```
┌─────────────────────────────────┐
│                                 │
│         [插画/图标 64px]         │
│                                 │
│      还没有任何对话               │
│   在下方输入框开始第一次对话吧     │
│                                 │
│       [ 快速开始 ]              │
│                                 │
└─────────────────────────────────┘
```

**规则**：
- 图标使用品牌色 15% 透明度
- 标题 16px/500
- 描述 13px/secondary
- 引导按钮在描述下方 16px
- 按钮使用品牌色 outline 样式

### 5.3 错误恢复模式

```
优先级1: 自动重试（指数退避，最多3次）
优先级2: Toast 通知 + "重试"按钮
优先级3: 内联错误提示（表单验证等）
优先级4: 回退到 Mock/缓存数据 + 提示徽章
```

```

│   ├── pages/
│   │   ├── chat.css            # 对话页专属样式
│   │   ├── workspace.css       # 工作室页
│   │   ├── plaza.css           # 广场页
│   │   ├── memory.css          # 记忆页
│   │   ├── connect.css         # 连接页
│   │   └── settings.css        # 设置页（含看板/Kanban/许可证/偏好/关于）
│   └── utilities.css           # 工具类

│   ├── niuma-chat-bridge.js    # 对话桥接
│   ├── theme-manager.js        # 主题管理
│   └── pages/
│       └── *.js                # 各页面专属逻辑
└── index.html                  # SPA 入口
```

### 6.2 CSS 方法论

采用 **Token-Driven BEM**：

```
.block {}
.block__element {}
.block--modifier {}
```

所有颜色必须通过 Token 引用，**禁止裸色值**出现在组件样式中。

```css
/* ✅ 正确 */
.card {
  background: var(--color-bg-card);
  border: 1px solid var(--color-border-subtle);
}

/* ❌ 禁止 */
.card {
  background: #131928;
  border: 1px solid #1a2540;
}
```

### 6.3 CSS 加载顺序

```html
<!-- 1. Token 系统（必须最先） -->
<link rel="stylesheet" href="css/design-tokens.css">

<!-- 2. 组件库 -->
<link rel="stylesheet" href="css/components.css">

<!-- 3. App 布局 -->
<link rel="stylesheet" href="css/layout-app.css">

<!-- 4. 页面样式（按路由懒加载） -->
<link rel="stylesheet" href="css/utilities.css">
```

## 七、主题切换实现

### 7.1 三按钮组件

```html
<div class="theme-toggle" role="radiogroup" aria-label="主题切换">
  <button class="theme-toggle__option" data-theme="day" aria-checked="false">
    <svg>...</svg> 浅色
  </button>
  <button class="theme-toggle__option" data-theme="night" aria-checked="false">
    <svg>...</svg> 深色
  </button>
  <button class="theme-toggle__option" data-theme="system" aria-checked="true">
    <svg>...</svg> 系统
  </button>
</div>
```

### 7.2 行为逻辑

```
默认：跟随系统（data-theme 不设置）
用户点击"浅色"：设置 data-theme="day"，存储 localStorage
用户点击"深色"：设置 data-theme="night"，存储 localStorage
用户点击"系统"：移除 data-theme，清除 localStorage
```

### 7.3 位置

- 桌面端：底部设置面板内的首选项
- 不放在顶部导航栏（保持导航极简）
