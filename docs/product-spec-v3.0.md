## 一、产品定位

### 1.1 一句话定位

> **打开 → 说话 → 得到结果。60岁大妈都能用的 AI 工作台。**

### 1.2 产品叙事

超级牛马是一个**本地优先的 AI Agent 协作工作台**。核心理念是让 AI Agent 像"牛马"一样高效干活，而用户不需要理解复杂的模型选择、Token预算、降级链等技术概念。

**三层终极形态：**

```
第三层 · 道（2030+）
  全智能网络的底层自运行规则
  具身机器人、脑机接口、虚拟世界——万亿智能体的协调法则

第二层 · 元大脑（2028+）
  所有模型的中枢神经
  大模型公司、硬件公司、应用——必须装太极引擎才能发挥最大价值

第一层 · 引擎（现在）
  智能路由器 + Token管控 + 自我进化
  选模型、管Token、自己修Bug、越用越聪明
```

### 1.3 核心叙事

> 太极引擎就是整个世界的大脑。
>
> 大模型是神经元，但它们之间没有突触。
> 太极引擎是突触——是让万亿个神经元变成一颗大脑的那一层。
>
> 不做文本生成模型，不让用户被锁在任何一家公司的生态里。
> 用户几十年的经验和直觉，变成太极引擎的行为模式，
> 让 AI 里住着你。

### 1.4 产品定位：AI 工作室（Studio），不是 AI 工厂（Factory）

| 维度 | 竞品（Coze/Dify） | 超级牛马 |
|:---|:---|:---|
| 模式 | 工厂模式——批量生产 AI Bot | 工作室模式——人与 Agent 协作产出 |
| 目标用户 | 开发者/企业搭建 Bot | 写作者/开发者/分析师/小白 |
| 交互方式 | 可视化编排工作流 | 对话驱动 + Agent 自主协作 |
| 记忆体系 | 单工具内对话记忆 | 跨所有AI、跨所有时间、永远不断 |
| 模型立场 | 锁定用户到自家生态 | 不属于任何模型公司——用户选择 |
| 进化方式 | 新版本→旧数据丢失 | "自我"持续积累，永不重置 |
| 部署方式 | 云端优先 | 本地优先 + 可选云端 |

### 1.5 目标

- **用户**：打开就能用，第1秒就知道怎么用，不读文档完成80%任务
- **技术**：8GB笔记本流畅运行，首屏加载<1s，零配置启动
- **商业**：免费试用3天，会员制付费，用户自助购买

## 二、NABC 价值分析

### N——需求（Need）

| 用户痛点 | 频次 | 严重度 |
|:---|:---:|:---:|
| 模型选择困难——不知道哪个模型适合什么任务 | 每次使用 | 高 |
| Token 浪费——复杂任务用了轻量模型，简单任务用了昂贵模型 | 每次对话 | 高 |
| 上下文碎片——切换工具后 AI 不认识你了 | 跨工具 | 中 |
| 学习成本高——要理解 "Agent""Workflow""RAG" 等概念 | 首次使用 | 中 |
| 数据隐私——不想把私人数据上传到云端 | 持续 | 高 |

### A——方案（Approach）

- **太极引擎 Smart Allocator**：自动分析任务复杂度，匹配最佳模型
- **Token 预算管控**：四两拨千斤，精准用力
- **Hermes Agent 记忆中枢**：跨工作间、跨会话的持久记忆
- **对话式交互**：零概念学习，打开即用
- **本地优先架构**：数据不出本地，可选云端

### B——收益（Benefits）

| 利益相关者 | 收益 |
|:---|:---|
| 写作者 | 专注创作，不用管技术细节，每条对话省30%时间 |
| 开发者 | 本地代码安全，自动模型路由，省Token成本 |
| 分析师 | 跨会话上下文保持，不需要反复喂背景 |
| 小白用户 | 打开就能用，3步完成首次对话 |

### C——竞争（Competition）

| 竞品 | 优势 | 弱点 | 我们怎么赢 |
|:---|:---|:---|:---|
| Coze | Bot 市场生态好 | 绑定豆包模型，云端强制 | 本地优先 + 模型自由 |
| Dify | 企业级 RAG 方案 | 配置复杂，学习成本高 | 零配置 + 对话驱动 |
| Kimi Work | 对话体验好 | 无 Agent 编排 | Agent 协作 + 太极引擎 |
| ChatGPT | 品牌认知强 | 云端强制，高价 | 本地数据 + Token优化 |
| Cursor | 代码场景优秀 | 仅限编程 | 全场景 AI 工作台 |

## 三、用户画像与场景

### 3.1 目标用户

| 用户 | 场景 | 核心需求 | 使用频次 |
|:---|:---|:---|:---:|
| **写作者** | 网文/自媒体/报告 | 不想管模型，写了就能得到好内容 | 每日3-5h |
| **开发者** | 编程/调试/架构 | 代码高亮、多文件上下文、终端接入 | 每日8h |
| **分析师** | 竞品/数据/研究 | 搜索、对比、结构化输出 | 每日2-4h |
| **普通小白** | 日常问答/翻译/总结 | 打开就能用，不学任何概念 | 每日0.5-1h |

### 3.2 用户旅程（首次使用）

```
Step 1 — 打开应用（5秒）
  下载安装 → 启动 → 看到对话界面

Step 2 — 首次对话（30秒）
  输入第一条消息 → 自动选择模型 → 看到回复

Step 3 — 探索能力（3分钟）
  尝试不同任务 → 发现自动路由智能 → 感受记忆持久

Step 4 — 深入使用（1天）
  创建工作间 → 配置项目Agent → 体验群组协作

Step 5 — 依赖形成（1周）
  习惯自主运行 → 发现越用越强 → 成为日常工具
```

### 3.3 关键体验指标

| 指标 | 目标 |
|:---|:---:|
| 首次对话完成时间 | ≤ 60秒 |
| 零配置可用度 | 100%（无需手动配置即可首次对话） |
| 对话回复首字时间 | ≤ 2秒 |
| 页面切换响应 | ≤ 100ms |
| 用户次日留存 | > 40% |

## 四、11条产品铁则

> 任何产品决策的最终评判标准。优先级高于一切。

| # | 铁则 | 描述 | 前端落点 |
|:--|:--|:---|:---|
| 1 | **极致安全** | 数据不出本地 + 不可被破解 | 无后端敏感数据传输，本地存储加密 |
| 2 | **极致性能** | 本地友好，8GB笔记本流畅运行 | 首屏加载<1s，构建产物<500KB |
| 3 | **极致轻量** | 大师50行解决的问题不写100行 | 零额外依赖，纯Vue3+CSS |
| 4 | **极致高效** | 走最优路径、最快速度 | 路由懒加载，SSE流式渲染 |
| 5 | **极致简单** | 60岁大妈都能用 | 默认Auto模式，零配置即可用 |
| 6 | **极致质量** | 产品×开发×产出物三重质量 | 完整的 TypeScript 类型覆盖 |
| 7 | **极致审美** | 看得见和看不见的都美 | 5色克制色板×充足留白×三主题 |
| 8 | **极致体验** | 用了就不想换别的平台 | 弹窗最少化，能toast的不弹窗 |
| 9 | **极致交互** | 超简单、超友好、超好用 | 引导流程≤60秒，3步完成首次对话 |
| 10 | **极致Token低消耗** | 国产模型优先，能省则省 | 前端不做Token展示，向用户隐藏复杂性 |
| 11 | **极致克制（Pi原则）** | 该做的一件不少，不该做的一件不多 | 不堆功能入口，用户不需要的不展示 |

## 五、太极引擎七律

太极引擎（Taiji Engine）——52模块的七元自进化系统，实现七律产品哲学。

| 定律 | 哲学 | 引擎模块 | 实现度 |
|:--|:--|:--|:--:|
| 阴阳平衡 | 本地←→云端+太极网格动态均衡 | Dynamic Balancer + TaijiMesh | 80% |
| 四两拨千斤 | 精准用力，不烧Token | Smart Allocator + CCR 压缩层 | 90% |
| 以静制动 | 关键时刻才说话 | Attention Engine + 夜巡 | 85% |
| 无为而治 | 自动修复，无需干预 | Self-Healing + 递归自进化 | 85% |
| 顺势而为 | 借力开源生态 | Leverage Arch + Hermes适配器 | 80% |
| 刚柔并济 | 硬开关+软积累 | Dual-Track + 能力开关 | 85% |
| 生生不息 | 越用越强，永续进化 | GoalLoop + 涌现引擎 + 递归进化 | 80% |

**当前综合实现度：约 83%（截至 2026-07-01）**

### 5.1 Hermes Agent 基座

超级牛马基于 **Nous Research Hermes Agent v0.16.0**（MIT开源）。Hermes Agent 是平台的**唯一 Root Node**——不是"一个功能模块"，而是整个工作台的**操作系统级基座**。

**Hermes 职责：**
- 平台大管家（BBG）：预装运行，用户可自定义名字
- Root Node：接收用户指令→分解意图→编排 Worker Agent 执行
- 记忆中枢：所有 Sub-Agent 的记忆由 Hermes 集中管理
- 技能分发：Hermes 统一管理技能库
- 全平台穿透：可查看任何工作间，但仅在用户明确要求时行使
- 被动行使：平时沉默，只在7种特定情况主动开口

**太极引擎与 Hermes 的演化关系：**

| 阶段 | 关系 | 描述 |
|:--|:--|:---|
| 当前 v1.x | **寄生** — 太极引擎 ⊂ Hermes Agent | 太极引擎住在 Hermes 的壳里 |
| v2.0 | **共生** — 太极引擎 ↔ Hermes Agent | 太极引擎独立运转 |
| v2.5+ | **容纳** — 太极引擎 ⊃ Hermes Agent | 太极引擎是壳，Hermes 变成可选运行时 |

## 七、全局设计系统

### 7.1 设计调性

**三个关键词：呼吸感、青春感、信任感。**

```
Soft Tech 风格——不是冷冰冰的黑底绿字终端，不是花里胡哨的渐变卡片。
是干净的、克制的、让人愿意每天打开用的工具。

调性对标：Linear 的克制 × Notion 的亲和 × Arc 浏览器的轻盈。
```

### 7.2 三层 Token 架构（DS v18.0）

```
L0 Theme Seeds ─── 三组种子变量（Night / Day / Classic）
    ↓ 自动派生
L1 Semantic Tokens ─── 语义化变量（--color-*, --text-*, --shadow-* 等）
    ↓ 组合引用
L2 Component Tokens ─── 组件级变量（--btn-*, --table-*, --modal-* 等）
```

**切换主题只改 L0 种子，全系统自动派生。**

### 7.3 品牌色

| 属性 | 值 | 备注 |
|:---|:---|:---|
| 品牌色 | `#4DA8F0` | 牛马蓝——唯一品牌强调色 |
| hover | `#6BBFFC` | 亮蓝 |
| muted | `rgba(77,168,240,0.08)` | 透明态 |
| glow | `rgba(77,168,240,0.10)` | 发光态 |

> **严格禁用**：粉紫色系（#E0B0FF, #D8B4FE, #C084FC, #A855F7 及其衍生）、emoji（全SVG图标）、渐变背景（限于品牌色单色使用）。

### 7.4 三主题引擎

| 主题 | 标识 | 氛围 | 默认 |
|:---|:---|:---|:---:|
| 夜间 Night | `data-theme="night"` | 深色沉浸，护眼 | ✅ 默认 |
| 日间 Day | `data-theme="day"` | 明亮清爽，办公 | |
| 经典 Classic | `data-theme="classic"` | 暖米纸本感，写作 | |

主题切换通过修改 `<html>` 标签的 `data-theme` 属性实现，CSS 变量自动响应。持久化到 `localStorage`。

### 7.5 布局体系

**4列工作区布局：**

```
┌────────┬───────────┬────────────────────────┬──────────┐
│NavRail │  Sidebar  │     Main Content        │  Config  │
│ 56px   │ 260-320px │     flex 1              │ 320px    │
│        │           │                         │          │
│ 图标   │ 列表/导航 │    核心工作区            │ 配置面板 │
│ 导航   │           │                         │ (可选)   │
└────────┴───────────┴────────────────────────┴──────────┘
```

- NavRail：固定56px，垂直图标导航，悬停弹出tooltip
- Sidebar：可调宽度（260-320px），响应隐藏
- Main Content：弹性宽度，页面主要内容
- Config：可选右侧面板，对话上下文/连接器配置
