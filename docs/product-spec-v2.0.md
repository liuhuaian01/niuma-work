### 底层基座：Hermes Agent

> 超级牛马基于 **Nous Research Hermes Agent v0.16.0**（MIT 开源，170+ 贡献者，542 PR 的工业级 Agent 框架）。

Hermes Agent 是平台的**唯一 Root Node**——它不是"一个功能模块"，而是整个工作台的**操作系统级基座**。

**Hermes 的职责：**
- **平台大管家（BBG）**：预装运行，用户可自定义名字
- **Root Node**：接收用户指令 → 分解意图 → 编排 Worker Agent 执行
- **记忆中枢**：所有 Sub-Agent 的记忆由 Hermes 集中管理，不独立存储
- **技能分发**：Hermes 统一管理技能库，Sub-Agent 使用 Hermes 分发的技能
- **全平台穿透**：可查看任何工作间、与任何 Agent 对话、访问所有文件——但仅在用户明确要求时行使
- **被动行使**：平时沉默观察，只在 7 种特定情况主动开口（破冰/进度/预警/错误/总结/模式发现/建议）

**太极引擎与 Hermes 的关系演化：**

| 阶段 | 关系 | 描述 |
|:--|:--|:--|
| 当前 | **寄生** — 太极引擎 ⊂ Hermes Agent | 太极引擎住在 Hermes 的壳里 |
| v2.0 | **共生** — 太极引擎 ↔ Hermes Agent | 太极引擎独立运转，Hermes 作为默认运行时 |
| v2.5+ | **容纳** — 太极引擎 ⊃ Hermes Agent | 太极引擎是壳，Hermes 变成可选运行时之一 |

**不取代 Hermes——反转嵌套关系。** 当 Hermes 的架构假设开始限制太极引擎进化时，自研 Taiji Native Runtime。

**Sub-Agent 体系：**
- Sub-Agent 在工作间内运行，独立上下文/终端/RPC
- 记忆和技能由 Hermes 集中管理——不做独立本地存储
- 工作间沙盒物理隔离——Agent 间不可直接跨工作间通信
- 跨工作间通信必须经 Hermes 中转，不可绕过
- Swarm 拓扑：Root(Hermes) → Worker(Sub-Agent) → Gate Validator → Gate Synthesizer

## 一、11条铁则 —— 任何决策的最终评判标准

> 违反任意一条，视为设计 Bug。不做优先级排序——全部平行，全部最高优先级。

### 1. 极致安全——双向安全

**对用户：** 默认本地处理，云端模型可选但需用户知情同意。Pi 准则覆盖率 100%。API Key 本地加密存储。使用云端模型时对话内容会离开本机——用户在引导流程中会收到明确告知。

**对开发者：** 产品不可被破解、蒸馏、复制。尤其是底层架构和产品哲学思维。代码可以开源，但太极引擎的"道"不可复制。

### 2. 极致性能——不吃设备

对本地友好。Gemma-4 能在 8GB 内存的笔记本上流畅运行。执行工作快，不卡顿。不是要求用户买 H100。

### 3. 极致轻量——大师级代码

以最优方案实现能力。高质量代码——大师 50 行解决的问题，不写 100 行。不堆模块，不加冗余。每一行代码都有存在的理由。

### 4. 极致高效——最优路径

整个平台工作走最优路径、最快速度。pre_chat_check 不串行调 6 个模块。消息链路不过 12 层。每一步都是必要的，每一步都是最快的。

### 5. 极致简单——60岁大妈能用

不需要复杂理解。不要求深度专业。打开 → 说话 → 得到结果。用户不需要知道"Token预算""降级链""能力开关"这些词。模型选择默认 Auto，用户只需说话。

### 6. 极致质量——三重质量

**产品质量：** Bug 少、不崩、不卡。
**开发质量：** 代码可读、可维护、可测试。
**产出物质量：** Gate 门禁确保 AI 输出不是垃圾。

### 7. 极致审美——全局审美

不只是看得见的 UI。看不见的——代码结构、API 设计、模块命名——都有审美。5 色克制色板 × 优雅接口设计。前后端一致的美学标准。

### 8. 极致体验——粘性第一

用户用了就不想换别的平台。不是功能多，是"刚好需要的东西刚好在"。反馈流畅，无打断感。

### 9. 极致交互——超简单超好用

弹窗最少化。操作最直接化。Auto 模式默认——用户不需要选模型。所见即所得。

### 10. 极致Token低消耗——能省则省

国产模型优先。上下文压缩率 50%+。Smart Allocator + TokenBudget + Savings 全线到位。每一分钱花在刀刃上。不是省成本，是让 Agent 能持续运行。

### 11. 极致克制（Pi原则）——双重克制

**面向开发者：** 该做的一件不少，不该做的一件不多。不符合产品定位和未来形态的事一律不做。

**面向太极引擎和用户：** 工作台执行任务时，不必要的事不做。大模型不调多余调用。克制是最高级的设计。

## 三、商业化与防破解

### 3.1 收费策略

我们不卖模型 API——用户自己向模型公司买 Key，和我们无关。

我们卖的是这个工具本身。

**主流程——用户自助：**

```
下载 → 免费试用 3 天 → 到期 → 充钱购买会员 → 自动激活 → 继续用
```

| 套餐 | 时长 |
|:--|:--|
| 月度 | 1 个月 |
| 季度 | 3 个月 |
| 年度 | 12 个月 |

**辅助流程——后台 Key：**

刘淮安后台可生成激活 Key（终身/1月/3月/半年/1年），用于：
- 内测用户赠送
- 合作伙伴授权
- 特殊情况处理

主路径是用户自己充钱自己激活。后台 Key 是辅助工具。

### 3.2 防破解 · 防蒸馏 · 防复制

**代码可以开源，但有一样东西你拿不走。**

太极引擎的价值不只在代码里。在三个你复制不了的东西里:

| 护城河 | 为什么复制不了 |
|:--|:--|
| **用户认知数据** | 100 万用户的"认知签名"——每个人的思维模式、偏好、决策习惯。这不是代码，是数据。你没用户就没人数据。 |
| **进化回传网络** | 那么多用户的引擎在自我进化——发现的模式、自调的参数、验证过的规则——全部汇聚到一起。你没网络就没进化。 |
| **"道"的积累** | 太极引擎运行了三年处理了数以亿计的任务——它的行为模式不是写出来的，是长出来的。你可以把代码抄走，但三年长出来的东西你长不出来。 |

**技术层面的防护:**

- **核心调度逻辑闭源** —— SmartAllocator 的力点探测算法、TokenBudget 的平衡策略、EngineWatchdog 的熔断规则——这些不放在开源仓库
- **引擎身份绑定** —— 启动时验证数字签名，非法副本拒绝运行
- **回传数据加密** —— 所有进化回传数据端到端加密，中间人无法提取元数据
- **模型权重不受影响** —— 我们不训练模型，所以不存在"权重被蒸馏"的问题

**战略层面的防护:**

- **快** —— 跑得比任何模仿者都快。他们还在抄 v1.6 的时候，你已经到 v3.0 了
- **网络效应** —— 用户越多，引擎越聪明。模仿者从零起步，没有用户数据，引擎是傻的
- **认知壁垒** —— 太极引擎的"道"不是代码。是刘淮安对 AI 世界的底层认知。你抄不了别人的脑子

## 五、架构原则

### 4.1 永不违反

1. **不做文本生成模型** — 不训练、不微调、不烧显卡。不是不想做，是资源有限，现阶段做不过大公司。但不代表太极引擎不能成为"模型"——它只是处于更高维度：不是生成文字的模型，是**编排模型的模型**。终局时，太极引擎本身就是最强大的元模型。
2. **不锁用户** — 任何模型可用、任何时间可换、数据随时可导出
3. **不碰内容** — 回传只传匿名元数据、对话绝不上传、隐私是底线
4. **过度克制** — Pi准则兜底，不该做的一律不做
5. **极致简单** — 用户不需要理解引擎，只需要感知好用

### 4.2 技术约束

- 预装两个本地模型（Gemma-4 + Qwen-VL）——零配置即可用
- 云端模型由用户自行配置 API Key
- 模型选择默认 Auto——Smart Allocator 自动推荐
- 所有引擎模块必须 try/except 保护 + 熔断兜底
- 任何模块挂了都不能影响消息发送
- 优雅关闭必须 flush + dispose——不丢数据

### 4.3 发布前检查清单

每版发布前，逐条对照 11 铁则：

- [ ] 极致安全 —— 用户数据安全 + 产品IP安全
- [ ] 极致性能 —— 低配设备也能跑
- [ ] 极致轻量 —— 没有一行冗余代码
- [ ] 极致高效 —— 消息链路是最短路径
- [ ] 极致简单 —— 60 岁大妈能完成首次对话
- [ ] 极致质量 —— 不交半成品，Gate 有真判断
- [ ] 极致审美 —— 代码和UI都讲究
- [ ] 极致体验 —— 用了不想换
- [ ] 极致交互 —— Auto 默认，零学习成本
- [ ] 极致Token —— 省钱可感知
- [ ] 极致克制 —— 没多加任何一个不该加的模块
