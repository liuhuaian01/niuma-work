## 一、产品定位与设计哲学

### 1.1 产品一句话

> 打开 → 说话 → 得到结果。60 岁大妈都能用的 AI 工作台。

### 1.2 设计铁则（与 11 条产品铁则对齐）

| 铁则 | 前端落点 |
|:--|:--|
| **极致简单** | 用户不需要知道"Token预算""降级链""能力开关"。默认 Auto 模式，零配置即可用 |
| **极致交互** | 弹窗最少化，操作最直接化。能不弹的用 toast，能一步的不两步 |
| **极致审美** | 5 色克制色板 × 系统字体栈 × 充足留白 × 0 粉紫色 |
| **极致体验** | 引导流程 ≤ 60 秒完成。第一条对话从打开到回复 ≤ 3 步 |
| **极致轻量** | 首屏加载 < 1s，页面切换无闪烁。不做重型动画 |
| **极致克制** | 不堆功能入口。用户不需要的，不展示 |

### 1.3 视觉调性

**三个关键词：呼吸感、青春感、信任感。**

```
Soft Tech 风格——不是冷冰冰的黑底绿字终端，不是花里胡哨的渐变卡片。
是干净的、克制的、让人愿意每天打开用的工具。

调性对标：Linear 的克制 × Notion 的亲和 × Arc 浏览器的轻盈。
```

## 二、用户画像

### 2.1 目标用户

| 用户 | 场景 | 核心需求 |
|:--|:--|:--|
| **写作者** | 网文/自媒体/报告 | 不想管模型，写了就能得到好内容 |
| **开发者** | 编程/调试/架构 | 需要代码高亮、多文件上下文、终端接入 |
| **分析师** | 竞品/数据/研究 | 需要搜索、对比、结构化输出 |
| **普通小白** | 日常问答/翻译/总结 | 打开就能用，不学习任何概念 |

### 2.2 用户期望

- 打开第 1 秒就知道怎么用
- 不读文档也能完成 80% 的任务
- 不被技术概念轰炸
- 界面不丑——愿意推荐给朋友

## 四、全局设计系统

### 4.1 色板（5 色克制体系）

```css
/* ===== 品牌色 ===== */
--brand:        #10B981;   /* 薄荷绿——唯一品牌强调色 */
--brand-hover:  #059669;
--brand-light:  #D1FAE5;   /* 极浅薄荷绿——hover 背景 */
--brand-subtle: #ECFDF5;   /* 微妙薄荷——选中态 */

/* ===== 中性色（灰度） ===== */
--bg-primary:   #FAFAFA;   /* 主背景——暖灰白 */
--bg-secondary: #F5F5F5;   /* 次背景——侧栏等 */
--bg-tertiary:  #FFFFFF;   /* 卡片/输入框 */
--bg-hover:     #F0F0F0;   /* 悬停态 */

--border:       #E5E5E5;   /* 分割线/边框 */
--border-light: #F0F0F0;   /* 微边框 */

--text-primary:   #1A1A1A; /* 主文本 */
--text-secondary: #6B7280; /* 次要文本 */
--text-tertiary:  #9CA3AF; /* 辅助文本/占位符 */
--text-inverse:   #FFFFFF; /* 反色文本 */

/* ===== 功能色 ===== */
--green:  #22C55E;  /* 成功/通过 */
--yellow: #F59E0B;  /* 警告/注意 */
--red:    #EF4444;  /* 错误/危险 */
--blue:   #3B82F6;  /* 信息/链接 */

/* ===== 严格禁用 ===== */
/* ❌ 粉紫色系（#E0B0FF, #D8B4FE, #C084FC, #A855F7 及其衍生）*/
/* ❌ 渐变背景（限于品牌色单色使用）*/
/* ❌ 超过 5 种颜色的页面 */
```

```css

--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);

/* 强制关闭动画（用户偏好） */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

- stroke-width：1.5px
- 颜色继承：`currentColor`

```
┌──────────────────────────────────────────────────────────┐
│ [侧栏 56px] │                                            │
│             │  ┌─────────────────────────────────────┐  │
│   💬        │  │  你好，我是 Hermes。                  │  │
│   🏠        │  │  你的 AI 工作助理。有什么可以帮你？   │  │
│   📋        │  │                                     │  │
│   🏪        │  │  [用户消息]                          │  │
│   🧠        │  │  [AI 回复 · Markdown · 代码高亮]     │  │
│   🔌        │  │                                     │  │
│   🏢        │  └─────────────────────────────────────┘  │
│   👤        │  ┌─────────────────────────────────────┐  │
│             │  │ [输入框]              [Auto ▾] [▶]  │  │
│   [头像]    │  └─────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

#### 5.1.2 首次引导（新用户首次打开）

流程：**3 步，≤ 60 秒。**

```
Step 1 — 欢迎（自动弹出，5 秒可跳过）
  "Hi，我是 Hermes 👋"
  "超级牛马是你的 AI 工作台。接下来 3 天完全免费。"
  [开始使用] 按钮

Step 2 — 问个好（预填 "你好，帮我写一个..."）
  用户输入第一句话 → Enter 发送
  Auto 模式自动选择最佳模型

Step 3 — 收到回复
  对话开始。引导完成。
  此后不再出现引导。
```

#### 5.1.3 消息气泡

| 角色 | 样式 | 位置 |
|:--|:--|:--|
| **用户** | 白底 + 1px 边框，右对齐 | 右 |
| **Hermes（系统）** | 无背景，左对齐，小字灰色 | 左 |
| **AI Agent** | 无背景，左对齐，Markdown 渲染 | 左 |
| **错误/中断** | 浅红底 + 红色左边框，重试按钮 | 居中 |

**Agent 消息额外显示：**
- 模型名称（灰色小字，hover 显示详情）
- Token 消耗（仅在用户主动查看时显示——不默认展示）
- 产出物附件（代码块/文件，可下载）

#### 5.1.4 输入区域

```
┌─────────────────────────────────────────────────┐
│ [@Agent] [📎] [🔍]              [Auto ▾] [→]   │
├─────────────────────────────────────────────────┤
│ 输入你想做的事...                                │
│ Shift+Enter 换行 · Enter 发送                   │
└─────────────────────────────────────────────────┘
```

**工具栏（水平排列，图标按钮）：**
- `@` — 提及 Agent（弹出 Agent 选择器）
- `📎` — 上传文件（拖拽也支持，拖入区域高亮）
- `🔍` — 搜索增强（开启联网搜索，显示绿点）

**模型选择器（Auto ▾）：**
- 默认显示 "Auto" — 绿色小胶囊
- 点击展开下拉：Auto（推荐）| DeepSeek V4 | Gemma-4 | Qwen-VL | 更多...
- 选中非 Auto 模型后胶囊变为模型名称
- 配额耗尽的模型灰显 + "已用完"标签

#### 5.1.5 SSE 流式响应

- AI 回复逐字出现（打字机效果，但速度 > 人眼跟踪速度——不要刻意慢）
- 代码块：流式完成后一次性渲染（避免语法高亮闪烁）
- 工具调用：折叠显示 "🔧 正在使用 xxx 工具..."
- 错误：消息气泡变红 + "重试" 按钮
- 中断：用户可随时点停止按钮（发送按钮在流式中变为 ■）

## 六、核心交互模式

### 6.1 Hermes 快速唤起

- **触发**：点击左下角头像，或 `Ctrl+Space`
- **行为**：从底部滑入对话气泡面板（宽 360px，高 400px）
- **定位**：右下角，距边缘 16px
- **内容**：直接对话，无需切换工作间
- **关闭**：点击外部区域，或 `Esc`

### 6.2 Toast 通知

```
位置：右上角，距顶部 16px
类型：
  ✅ 成功 — 绿色图标 + 简短文案，3s 自动消失
  ⚠️  警告 — 黄色图标 + 文案 + 操作按钮（可选），5s
  ❌ 错误 — 红色图标 + 文案 + "查看详情"，手动关闭
  ℹ️  信息 — 蓝色图标 + 文案，3s

同一时间最多显示 2 条 toast。
新 toast 出现时，旧的向上推移。
```

### 6.3 弹窗（Dialog）

```
使用原则：能不用就不用。满足以下至少 2 条才弹：
  1. 需要用户明确确认的操作（删除、重置）
  2. 需要用户输入额外信息
  3. 操作不可逆

样式：
- 居中，max-width: 480px
- 背景遮罩：rgba(0,0,0,0.3) + backdrop-blur(4px)
- 弹窗本身：白底 + 圆角 12px + shadow-dialog
- "取消" 按钮在左（灰色），"确认" 在右（品牌色）
- 点遮罩可关闭（除非标记为 "必须选择"）

禁止：
- 弹窗中再弹窗
- 超过 2 个按钮的弹窗
- 超过 200 字的弹窗内容
```

### 6.4 右键菜单（Context Menu）

```
- 消息气泡右键：复制 / 重新生成 / 删除
- 工作间右键：重命名 / 复制 / 删除
- Agent 右键：编辑 / 暂停 / 删除
```

### 6.5 拖拽

```
- 文件拖入输入框区域 → 自动上传
- 工作间卡片可拖拽排序
- 拖拽时：原位置半透明 + 插入位置蓝色指示线
```

### 6.6 键盘快捷键

| 快捷键 | 功能 |
|:--|:--|
| `Ctrl+Space` | 唤起 Hermes |
| `Ctrl+K` | 全局搜索/命令面板 |
| `Ctrl+N` | 新建对话 |
| `Ctrl+Enter` | 发送消息 |
| `Shift+Enter` | 消息换行 |
| `Esc` | 关闭弹窗/Hermes/面板 |
| `Ctrl+1~8` | 切换导航页 |

## 七、状态与反馈系统

### 7.1 空状态

每个页面必须有高质量的空状态设计，不是"暂无数据"四个字。

| 页面 | 空状态设计 |
|:--|:--|
| 对话 | Hermes 欢迎语 + 3 个快捷提问建议 |
| 工作室 | "创建你的第一个工作间" + 3 个模板入口 |
| 任务 | "还没有任务。在对话中 @Agent 开始一个复杂任务" |
| 记忆 | "记忆会在你使用中自动积累" + 示意动画 |
| 连接 | "接入模型 API 或 MCP 工具来扩展能力" |
| 办公室 | "使用几天后这里会有你的用量数据" |

### 7.2 加载状态

```
骨架屏（Skeleton）：
- 消息列表加载中：灰色脉冲条，宽度随机 60%-90%
- 工作间卡片加载：3 个灰色矩形脉冲
- 不使用转圈圈 spinner（除非 < 1s 的操作）

进度条（Progress）：
- 模型下载：蓝色进度条 + 百分比 + 速度
- 文件上传：绿色进度条 + 文件名
- 大数据导出：进度条 + "预计 30 秒"
```

### 7.3 错误状态

```
全局错误（API 不可用）：
  ┌──────────────────────────┐
  │  😔                       │
  │  无法连接到服务            │
  │  请检查后台是否在运行       │
  │  [重试] [查看日志]        │
  └──────────────────────────┘

网络错误（消息发送失败）：
  消息气泡变红 + "发送失败" + [重试] 按钮

模型错误（模型不可用）：
  Toast："DeepSeek V4 当前不可用，已自动切换到 Gemma-4"
  输入框模型选择器更新
```

## 八、性能与兼容性

### 8.1 性能指标

| 指标 | 目标 | 测量方式 |
|:--|:--|:--|
| 首屏加载（FCP） | < 1.0s | 本地启动到可交互 |
| 页面切换 | < 100ms | 导航标签切换 |
| SSE 首字节 | < 500ms | 发送消息到第一个 token 出现 |
| 虚拟列表帧率 | 60fps | 10000+ 消息滚动 |
| 内存占用 | < 150MB | 运行 1 小时后 |
| 包体积 | < 50MB | WebView2 壳 + 前端资源 |

### 8.2 兼容性

```
- 平台：Windows 10/11（WebView2 Runtime）
- 最低硬件：8GB RAM，集成显卡
- 分辨率：最小 1024×768，推荐 1440×900
- 不支持移动端（v2.0 前）
- 不支持 macOS/Linux（v2.0 前）
- 无离线模式（v1.0 不下）
```

## 九、技术栈要求

### 9.1 框架选择

```
推荐：Vue 3 + Vite + TypeScript
备选：React 18 + Vite + TypeScript
禁止：jQuery / 原生 JS 单文件 HTML（原型阶段已结束）
```

### 9.2 必需依赖

```
UI 组件：不引入重型 UI 库（Ant Design/Element Plus 禁用）
  → 自行封装组件，保持轻量

Markdown 渲染：markdown-it + highlight.js（代码高亮）
图标：Lucide Icons
状态管理：Pinia（Vue）/ Zustand（React）
HTTP：fetch（原生，不引入 axios）
SSE：EventSource（原生）
虚拟列表：@tanstack/virtual
```

### 9.3 部署方式

```
- 前端构建产物嵌入 FastAPI 静态文件服务
- WebView2 加载 http://localhost:18080
- 开发时 Vite dev server 代理到 FastAPI
```

### 9.4 代码规范

```
- ESLint + Prettier
- 组件文件 ≤ 300 行
- 单文件组件（SFC）—— template + script + style
- CSS Variables 统一主题（不写硬编码颜色）
- 无 !important（除非覆盖第三方库）
```

## 十一、验收标准

### 11.1 功能验收

| # | 验收项 | 通过标准 |
|:--:|:--|:--|
| 1 | 首次引导 | 新用户 3 步完成引导，≤ 60 秒 |
| 2 | 对话 | 发送消息 → SSE 流式回复 → 渲染正确 |

| 7 | 空状态 | 所有页面空状态有引导性文案和操作入口 |
