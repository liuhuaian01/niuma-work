# Super Niuma Chat Interface v9 — 迭代分析报告

> 日期：2026-06-26 | 来源：Hermes Desktop 架构参考 + 超级牛马产品架构 + DESIGN-v3-youth

## 一、核心设计变更：对话列表双类型模型

### 产品逻辑映射

| 产品概念 | 对话列表呈现 | 视觉特征 |
|---------|------------|---------|
| 创建项目 → 拉多个 Agent | **群组对话** (项目群组) | 四宫格堆叠头像 + 成员数角标 + 项目名 |
| 创建单个 Agent | **单 Agent 对话** | 单体头像 + 角色标签 + 模型标签 |

### 区隔设计

```
┌─ 对话列表 ─────────────────────┐
│ 🔍 搜索...          [+] 新建  │
│ [全部] [项目群组] [单Agent] [已固定] │
│                                │
│ ▓▓ 项目群组 (4)          ▾    │
│ ┌──────────────────────────┐  │
│ │ ▦▦ 赛博工坊 — 主控制台   │  │ ← active, 5 agents
│ │ ▤▤ ▦▦ ▦▦ ▦▦     [+5]   │  │   四宫格堆叠
│ │ 📍 不良人 — 第一章创作    │  │ ← pinned, 3 agents
│ │ ...                       │  │
│ └──────────────────────────┘  │
│                                │
│ ▓▓ 单 Agent (8)          ▾    │
│ ┌──────────────────────────┐  │
│ │ ◉ 超级牛马   [大管家]    │  │ ← 单体头像 + 角色标签
│ │ ◉ 代码专家   [DS v4]     │  │ ← 单体头像 + 模型标签
│ │ ...                       │  │
│ └──────────────────────────┘  │
│                                │
│ ▓▓ 已归档 (3)           ▸    │ ← 默认折叠
└────────────────────────────────┘
```

## 二、从 Hermes Desktop 借鉴的前端改进

### ✅ 已采纳 (v9 原型已实现)

| Hermes 特性 | 超级牛马落地 | 优先级 |
|------------|-------------|:--:|
| **侧边栏搜索** | 搜索栏 (toggle显示，FTS5 后端) | P0 |
| **会话 Archive** | 已归档分区 (折叠态，灰色半透明) | P0 |
| **创建菜单** | "+" 下拉：新建Agent / 新建项目群组 / 新建工作间 | P0 |
| **筛选 Chips** | 全部 / 项目群组 / 单Agent / 已固定 / 活跃中 | P0 |
| **状态指示** | 在线(绿点+辉光) / 忙碌(黄点) / 离线(灰点) | P0 |
| **类型标记** | 角色标签(大管家/分析者) + 模型标签(DS v4/Kimi) | P0 |

### 🟡 规划中 (后续迭代)

| Hermes 特性 | 超级牛马适配方案 | 阶段 |
|------------|----------------|:--:|
| **Cmd+K 命令面板** | 全局命令中心：快速切换对话、创建Agent、搜索 | v1.6 |
| **模型模糊搜索** | 输入框底部模型选择器 → 点击展开模糊搜索面板 | v1.6 |
| **对话列表右键菜单** | Pin/Archive/Mark Read/Delete | v1.6 |
| **类型安全 i18n** | 先中文，v2.0 考虑 i18next | v2.0 |
| **会话拖拽排序** | Pin → 手动排序对话列表 | v1.7 |
| **快捷键导航** | Cmd+1~9 切换前9个对话 | v1.6 |

### ❌ 不适合借鉴

| Hermes 特性 | 原因 |
|------------|------|
| CLI/TUI 终端 | 超级牛马不需要终端界面 |
| Discord/Telegram 桥接 | 非 MVP 范围 |
| Docker-in-Docker | 不符合"极致轻量"铁则 |
| 技能市场繁重体系 | 太极引擎有自己的技能门控机制 |

## 四、交互增强细节

### 群组对话特有交互
1. **四宫格头像堆叠** — 最多显示4个Agent头像，超出部分用 `+N` 角标
2. **成员面板** — 右侧信息面板展示群组成员列表、状态、模型
3. **@提及** — 输入框中 @ 可提及群组内特定 Agent
4. **群组状态** — 在线(绿辉光脉冲) / 忙碌(黄点) / 离线(灰点)
5. **Pin 固定** — 固定群组带图钉图标，顶部优先显示

### 单 Agent 对话特有交互
1. **单体头像** — 圆角矩形 + 渐变背景 + 图标
2. **角色标签** — 大管家/分析者/创作者/构建者等角色pill
3. **模型标签** — DS v4 / Kimi / Gemini 等模型标识
4. **状态圆点** — 右下角在线状态指示器
