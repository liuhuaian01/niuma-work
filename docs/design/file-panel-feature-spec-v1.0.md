# Super Niuma 文件面板功能规格书 v1.0

> 基于用户研究 + 竞品分析，对齐11条产品设计铁则
> 2026-06-11 | 刘淮安 LIUHUAIAN

---

## 一、核心定位

**文件面板 = Agent 的操作台，不是文件浏览器。**

用户打开它不是来"逛文件"的——每次都有明确目的：找模型、查日志、看导出、清空间。所以不做 VSCode 那种通用文件树，做一个**语义驱动的任务型面板**。

### 一句话描述
> 让用户用"人话"（模型、导出、日志、知识库）找到文件，让 Agent 能直接在文件上干活。

---

## 二、设计铁则对齐（前11条逐条核查）

| # | 铁则 | 文件面板如何对齐 |
|---|------|-----------------|
| 1 | 极致安全 | 文件操作在本地沙箱内，不暴露文件系统路径；预览不写入临时文件 |
| 2 | 极致性能 | 语义分组视图零查询延迟（内存索引），搜索 <200ms，文件预览流式加载 |
| 3 | 极致轻量 | 不引入额外 Electron 模块，复用现有 WebView2 能力；一个面板不拆多页 |
| 4 | 极致Token低消耗 | 知识库文件预览走本地 RAG 索引摘要，不全量送入 LLM |
| 5 | 极致高效 | 右键菜单全键盘可达，搜索即导航，Agent 一键操作文件 |
| 6 | 极致简单 | 5个语义分组（模型/导出/日志/知识库/配置），奶奶看懂 |
| 7 | 极致质量 | 预览内核覆盖 Markdown/JSON/日志/图片/CSV 五种格式，每种完整闭环 |
| 8 | 极致审美 | 5色克制色板，文件图标精简（5种分组各一色），动画 60fps |
| 9 | 极致体验 | 单击预览 <100ms，右键菜单即时弹出，搜索 debounce 150ms |
| 10 | 极致交互 | 拖拽排序、右键Agent操作、搜索框即输即搜，所见即所得 |
| 11 | 极致克制 | 不做通用文件浏览器，不做 FTP/远程挂载，不做版本对比 |

## 三、架构：语义分组视图

### 3.1 为什么不用文件树

用户研究结论：**没有人想看到 `/data/models/qwen2.5-7b-q4.gguf` 这种路径。**

竞品分析印证：OpenHuman 放弃树形目录改扁平 Vault，Notion 用无限嵌套侧边栏但本质是"页面分组"而非"文件夹"。

我们的路线：

```
文件面板（单一面板，不拆多Tab）
├── 📦 模型        ← Glob: data/models/*.gguf, *.safetensors, config.json
├── 📤 导出        ← Glob: data/exports/*.md, *.json, *.html
├── 📋 日志        ← Glob: data/logs/*.log, *.txt
├── 📚 知识库      ← LanceDB 索引 + 文档元数据
└── ⚙️ 配置        ← Glob: data/config/*.json, *.yaml, *.toml
```

### 3.2 分组视图的交互模式

每个分组是一个**可展开的虚拟列表**（不是树形）：

```
┌─────────────────────────┐
│ 🔍 搜索文件...          │  ← 全局搜索栏，始终置顶
├─────────────────────────┤
│ 📦 模型 (3)      [排序] │  ← 分组头，右侧排序切换
│   qwen2.5-7b-q4    4.2G │  ← 列表项：名称 + 大小 + Agent操作入口
│   deepseek-v3-q8   8.7G │
│   config.json      1.2K │
├─────────────────────────┤
│ 📤 导出 (12)     [排序]  │
│   2026-06-10-对话总结.md │
│   ...                   │
└─────────────────────────┘
```

### 3.3 排序与筛选

每个分组独立支持：
- **排序**：按名称 / 修改时间 / 文件大小（默认按修改时间倒序，日志按时间倒序）
- **筛选**：文件名模糊匹配（分组内搜索）
- **全局搜索**：跨分组搜索，高亮命中项

## 四、核心功能矩阵

| 功能 | 优先级 | 描述 |
|------|:---:|------|
| 语义分组视图 | P0 | 5个固定分组，按用途而非路径聚合文件 |
| 文件搜索 | P0 | 全局+分组内搜索，即输即搜，高亮匹配 |
| 排序切换 | P0 | 名称/时间/大小，分组内独立记忆排序偏好 |
| 单击预览 | P0 | 侧面板或内联预览，支持 MD/JSON/日志/图片/CSV |
| 右键菜单 | P0 | 复制路径/用Agent打开/导出/删除，全键盘可达 |
| 磁盘概览 | P1 | 分组级别大小统计+总量，磁盘告警阈值提示 |
| Agent 操作入口 | P1 | 右键"发送给 Agent"、拖拽文件到对话区 |
| 批量操作 | P1 | 多选删除、批量导出、一键清理日志 |
| 知识库树形浏览 | P2 | LanceDB 文档按文件夹层级展开（保留知识库语义）|
| 拖拽排序 | P2 | 分组内文件拖拽到对话区或工作间 |

## 五、文件预览内核

### 5.1 支持的格式（v1.0）

| 格式 | 预览方式 | 备注 |
|------|---------|------|
| `.md` | Markdown 渲染器（复用现有） | 代码块语法高亮 |
| `.json` | 语法着色 + 折叠 | JSON5 容错解析 |
| `.log` / `.txt` | 纯文本 + 行号 | 最后1000行默认，支持滚动加载更多 |
| `.png` / `.jpg` / `.gif` | 图片预览 | 限制最大 5MB |
| `.csv` | 表格视图 | 前100行，支持列宽拖拽 |

### 5.2 预览交互

- **单击文件** → 右侧展开预览面板（占面板宽度 60%）
- **双击文件** → 全屏预览（模态层）
- **预览面板关闭** → 点击空白或 Esc
- **Agent 按钮** → 预览面板底部固定 "发送给 Agent" 按钮

## 六、竞品差异策略

| 维度 | VSCode/Cursor | Super Niuma |
|------|:---:|:---:|
| 组织方式 | 文件系统树 | 语义分组视图 |
| 主导交互 | 浏览→点击 | 搜索→预览→Agent操作 |
| Agent 集成 | 右键菜单（有限） | 文件即 Agent 操作对象 |
| 噪音过滤 | 手动 exlude | 默认屏蔽，语义分组天然过滤 |
| 知识库 | N/A | LanceDB 树形浏览 + 语义搜索 |
| 跨来源聚合 | 不支持 | 本地文件 + LanceDB + 第三方数据源（未来）|

**不做的事**：
- 不做 Git 状态装饰器（工作台不是 IDE）
- 不做文件夹新建/重命名（不是文件管理器）
- 不做 FTP/云存储挂载（保持极简）

## 七、交互细节规范

### 7.1 右键菜单

```
┌──────────────────────┐
│ 👁  预览             │  ← 默认项（加粗）
│ 📋  复制路径         │
│ 🤖  发送给 Agent     │  ← Agent 操作入口
│ 📤  导出...          │
│ ───────────────────  │
│ 🗑  删除             │  ← 危险操作（红色，需确认）
└──────────────────────┘
```

- **键盘**：右键菜单每项配有下划线快捷键字母
- **Agent 发送**：将文件路径 + 预览摘要注入当前对话（不入完整文件内容，走 Token 低消耗策略）

### 7.2 搜索交互

- **全局搜索栏**：面板顶部固定，`Ctrl+K` 聚焦
- **即输即搜**：150ms debounce
- **结果展示**：跨分组命中高亮，分组名右侧显示命中数
- **无结果**：显示"未找到文件，试试换个关键词？" + 创建新文件入口

### 7.3 空状态

```
┌─────────────────────────┐
│      📂                 │
│   这里还没有文件         │
│                         │
│  文件会在以下情况出现：   │
│  · 下载模型后           │
│  · 导出对话后           │
│  · 工作台运行产生日志    │
│  · 上传知识库文档后      │
└─────────────────────────┘
```

## 九、非功能需求

| 指标 | 目标 |
|------|------|
| 面板冷打开 | <200ms（分组列表内存缓存） |
| 搜索响应 | <200ms（本地 glob + 字符串匹配） |
| 文件预览首屏 | <100ms（流式加载首帧） |
| 大日志预览 | 支持 50MB 日志文件，首帧 <500ms |
| 内存占用 | 面板打开增量 <30MB |
