# Kimi Work Beta 真实用户评测汇总

> 汇总日期：2026-06-13 | 信源：CSDN实测、neican.ai深度评测、站长之家报道、搜狐报道
> 覆盖：2篇深度实测 + 2篇媒体评测

## 一、一句话总结

**Kimi Work 是当前市面上最强的本地 Agent 工作台，但它是一个"0→80分"的半自动工具，最后20分仍需要人类专业判断。**

## 二、用户实测数据

### CSDN 博主 30小时深度测试（发布当天上手）

| 测试任务 | 耗时 | 子Agent | 产出 | 评价 |
|------|------|:--:|------|------|

| 13h编码压测 | 13h18min | 47/300峰值 | 4386行代码，24文件，完整博客系统 | "可以直接上线的项目骨架" |


另一个用户：试用30小时后，**退订了 Claude Computer Use 订阅**。

## 三、用户报告的缺点（按严重程度排序）

### 🔴 P0 级问题

| 问题 | 详情 | 对超级牛马的启示 |
|------|------|------|
| **闭源生态深度绑定** | "Kimi模型策略调整或封号，工作流将受到不可逆影响"（neican.ai 原文） | 超级牛马多模型混合策略是正确方向 |
| **不是全自动方案** | "Kimi Work 擅长从0到80分，最后20分的人类直觉和专业判断还是得你自己来" | 验证太极引擎"人从决策者变成守门人"的定位 |

### 🟡 P1 级问题

| 问题 | 详情 | 对超级牛马的启示 |
|------|------|------|
| **Token消耗巨大** | 13h任务≈30万Tokens，免费额度仅3-4个任务/天 | Token优化是刚需，验证"极致Token低消耗"铁则 |
| **浏览器精度不足** | SPA页面点击坐标偏移约10px | MCP Playwright 可能是更好的浏览器接入方案 |
| **高负载噪音** | 47个子Agent时CPU>30%，M3 Max风扇"起飞去火星" | 超级牛马需关注本地推理的硬件友好性 |
| **文档排版缺陷** | PDF/Word导出有明显留白问题 | 超级牛马的文件导出需做格式校验 |
| **复杂调试仍需人工** | CSS响应式问题 Agent 尝试4次未解决 | "极致高效"≠"全自动"，人机协作是正确方向 |

### 🟢 P2 级问题

| 问题 | 详情 | 对超级牛马的启示 |
|------|------|------|
| **无跨设备同步** | 本地存储，无法云端同步 | 超级牛马三层记忆+云存储的差异化机会 |
| **内测门槛** | 需要申请资格，非完全公开 | Beta产品质量不稳定的风险 |
| **代码质量不均** | 13h项目中存在`any`类型滥用、内联CSS | 代码Sub-Agent需要代码规范注入 |
| **学习成本** | "需要用户自己探索和配置的工具平台" | 验证"极致简单"铁则——奶奶都能用 |

## 五、内测资格说明

来自多篇报道的交叉验证：
- Kimi Work 当前状态：**Beta版内部测试**（非公测）
- 登录可能受限于：**白名单/邀请制**

→ 这与 Kimi Work 官方定位不矛盾，但说明产品离"公开发布"还有距离。

## 六、对超级牛马的策略影响

### 确认正确的方向
1. **多模型混用 > 单一模型绑定** — 用户最大的担忧就是生态锁定
2. **记忆系统是护城河** — 所有评测报告无一提到记忆，Kimi Work 确实没有
3. **Token优化是刚需** — 300K Token/任务，付费压力巨大
4. **人机协作 > 全自动** — 即使是 Kimi Work，也只能做到"0→80分"

### 需要重新评估的方向
1. **Agent Swarm** — Kimi Work 的 300 Agent 并行是真实能力而非营销，超级牛马的 Swarm 拓扑需要加速落地
2. **浏览器操控** — Kimi Claw 让用户退订了 Claude Computer Use，这是真实的产品力
3. **任务→结果** — Kimi Work "你给我目标，我给你结果" 的交互哲学比超级牛马当前的对话模式更先进

### 超级牛马应该打的牌

```
Kimi Work 做不了的事 → 超级牛马做：

1. 跨会话记忆     → Kimi 完全缺失
2. 用户知识库RAG  → Kimi 只有通用数据
3. 模型自由切换   → Kimi 绑死自有模型
4. 长期项目沉淀   → Kimi 无持久化记忆
5. Linux 支持     → Kimi 仅 Mac/Windows
6. 数据安全可控   → Kimi 依赖云端 API
```
